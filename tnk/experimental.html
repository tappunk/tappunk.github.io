Experimental ​

tnk ships with llama (llama.cpp) as the stable default inference runtime. You can switch the runtime to mlxcel to run MLX models natively on Apple Silicon — this is a supported configuration but not the shipped default. The features below are truly experimental, not yet recommended for production workflows. They are not dependencies of tnk and do not ship with it.

Inference runtimes ​

tnk uses llama by default for GGUF models. For MLX-native models, switch the runtime to mlxcel:

bash
# Install mlxcel (separate from tnk)
brew install lablub/mlxcel/mlxcel

# Start with the default preset
tnk engine start --runtime mlxcel

Create a preset file in ~/.config/tnk/provider.d/ (.ini format):

ini
runtime = mlxcel
model   = ~/opt/models/mlx-community/Qwen3.5-9B-4bit
extra   = --max-batch-size 1

The model field can be a path to a local directory or a Hugging Face namespace/name. Use tnk download to fetch models.

tnk's focus is on llama and mlxcel as the supported inference runtimes.

Sandbox coding agents ​

tnk ships with pi as the default coding agent for sandbox provisioned workspaces.

Switching back to stable defaults ​

bash
# Revert engine to llama
tnk engine stop --all
tnk engine start --runtime llama
