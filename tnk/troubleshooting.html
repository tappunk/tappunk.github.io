Troubleshooting ​

sandbox shell says command not found for profile tools ​

Re-apply profile assets and reprovision:

bash
tnk init --force
tnk sandbox shell --profile pi --no-tty --command "command -v pi || true"

Sandbox cannot reach inference API ​

Validate host bind and endpoints:

bash
tnk config show
tnk engine status

The Lima host.lima.internal route reaches 127.0.0.1 on the host.

Running from wrong directory/context ​

tnk sandbox commands require project context under workspace_root.

If you are outside that root, tnk exits with usage-context errors.

Sandbox cleanup ​

List and stop all managed sandboxes:

bash
tnk sandbox ls
tnk sandbox stop --all

Delete uses current project context:

bash
cd ~/code/project-a && tnk sandbox delete --yes

Service plane issues ​

bash
tnk services status --output json
tnk services restart

Engine/process drift ​

bash
tnk engine stop --all
tnk engine start --runtime llama

Reading inference server logs ​

Logs live in ~/.cache/tnk/:

bash
# View the last 100 lines
tail -n 100 ~/.cache/tnk/llama-server.log

# Follow live output
tail -f ~/.cache/tnk/llama-server.log

# Check error log separately
tail -n 100 ~/.cache/tnk/llama-server-err.log

Inference server unreachable from sandbox ​

The Lima host.lima.internal route reaches 127.0.0.1 on the host. Check:

bash
tnk config show | grep bind_host
lsof -i :8080 -P -n

If the engine isn't listening on port 8080, restart it with tnk engine stop followed by tnk engine start --runtime llama.

Profile provisioning hangs ​

Provisioning hangs from unreachable package mirrors or a misconfigured host gateway.

bash
tnk sandbox shell --profile pi --no-tty --command "ping -c 3 github.com"
tnk config show | grep gateway

If DNS fails, the sandbox network is broken. Try tnk init --force to regenerate configs.

UID/GID drift on mounted files ​

Files created inside a sandbox appear on the host with the sandbox UID instead of your host UID. Check:

bash
ls -ln ~/code/myproject
id

If the file owner shows a numeric UID that does not match your id -u, tnk could not sync the user. Recreate the sandbox:

bash
tnk sandbox stop --name tnk-myproject
tnk sandbox shell

tnk rebuilds the tnk user with the correct host UID on the next creation.

Pre-flight diagnostics ​

bash
tnk doctor
