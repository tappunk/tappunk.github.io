On this page

Commands ​

Top-level ​

  • tnk engine — manage inference engine
  • tnk sandbox — manage project sandboxes
  • tnk services — manage persistent services runtime
  • tnk run — start engine + services
  • tnk shutdown — shutdown all managed components
  • tnk completion — generate shell completion scripts
  • tnk init — initialize tnk from upstream specs
  • tnk config — manage tnk config
  • tnk doctor — run diagnostics and health checks
  • tnk download — download models from Hugging Face Hub

Engine ​

Start ​

bash
tnk engine start --preset PRESET --runtime RUNTIME [--bind-host HOST] [--engine-server-port PORT] [--foreground]

Key flags:

  • --runtime engine runtime (llama by default). Experimental runtimes are described in Experimental.
  • --preset preset name (must match a .ini file in provider.d/)
  • --bind-host server bind address (127.0.0.1 default)
  • --engine-server-port override API port
  • --foreground run in foreground (blocking mode) instead of as a background daemon

Stop ​

bash
tnk engine stop --runtime RUNTIME [--all]

Default runtime is llama. --all stops every running engine regardless of runtime. Experimental runtimes are described in Experimental.

Status ​

bash
tnk engine status --output text|json|ndjson

Presets ​

bash
tnk engine presets --runtime RUNTIME --output text|json|ndjson [--strict]

Lists configured model presets from provider.d/. Use --strict to show only presets with an explicit runtime field matching the selected engine.

Sandbox ​

Start ​

bash
tnk sandbox start [--profile PROFILE] [--audit-log PATH] [--shell]

Run without --profile to use the default profile. Use --shell to attach an interactive shell after starting.

Shell ​

bash
tnk sandbox shell [--profile PROFILE] [-c|--command CMD] [--no-tty] [-e|--env KEY=VALUE] [--audit-log PATH]

Flags:

  • -c, --command execute a non-interactive command instead of opening a login shell
  • --no-tty bypass TTY requirements for automation
  • -e, --env add environment variables in KEY=VALUE form (repeatable)

Stop ​

bash
tnk sandbox stop [--all] [--name SANDBOX ...]

--all and --name are mutually exclusive.

Delete ​

bash
tnk sandbox delete [--yes] [--dry-run]

Targets the sandbox for the current project directory. -n, --dry-run previews the action without side effects. -y, --yes skips the confirmation prompt.

List ​

bash
tnk sandbox ls --output text|json|ndjson [--quiet]

--quiet (-q) outputs only sandbox names, one per line.

Services ​

Manages the tnk services VM — a dedicated sandbox that runs the MCP bridge and search tooling. This services VM is a core default, shared by all project sandboxes. Sandboxes reach these MCPs via the runtime environment contract without exposing them to the host or to other sandboxes.

  • tnk-services: MCP bridge and routing on port 18765
  • tnk-searxng: search tooling on port 18766

The services VM runs inside a Lima VM.

bash
tnk services start [--dry-run]
tnk services stop [--dry-run]
tnk services restart [--dry-run]
tnk services delete [--yes] [--dry-run]
tnk services status --output text|json|ndjson

Delete ​

bash
tnk services delete [--yes] [--dry-run]

Removes the services VM. Requires --yes to skip confirmation.

Run tnk with no subcommand to show engine, services, and sandbox status.

Run / shutdown ​

bash
tnk run [--preset PRESET] [--runtime RUNTIME] [--dry-run]
tnk shutdown [--timeout SECONDS] [--dry-run]

tnk run boots the engine and services runtime. tnk shutdown stops sandboxes, services, and engine in dependency order. Use --timeout to set the per-component grace period (default 30 seconds).

Global flags ​

--quiet (-q) and --verbose (-v) are available on all commands:

  • -q suppresses non-error informational output
  • -v shows detailed operational logs

Download ​

Download models from Hugging Face Hub. Accepts a Hugging Face URL, hf:// URI, or repo ID (namespace/name).

bash
tnk download <URL> --revision v1.0.0 --workers 4
tnk download hf://tappunk/tnk-model --dry-run
tnk download namespace/repo --force --output json

Options:

  • --revision <REVISION> custom revision (branch, tag, or commit)
  • --workers <WORKERS> max concurrent downloads (4 default)
  • --force overwrite existing files even if sizes match
  • -o, --output output mode (text, json, ndjson)
  • -n, --dry-run preview files without downloading
  • -q, --quiet suppress non-error operational output
  • -v, --verbose show detailed operational logs

Config / init / doctor / image ​

bash
tnk init [--git-url URL] [--force]
tnk config init [--force]
tnk config show
tnk doctor

Output contract ​

When available, output mode is explicit:

  • text: human-oriented status on stderr
  • json: structured payload on stdout
  • ndjson: one JSON object per line on stdout

Shell scripts stay stable. Terminal output stays readable.
